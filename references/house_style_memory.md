## What The User Actually Wants

The target is not a static PPT, not a subtitle video, and not flashy animation. The target is a Douyin-ready finance information-animation segment that can be stitched with a separate human/avatar video.

Codex normally owns the information-animation part only. Do not manage avatar or Jianying stitching unless the user explicitly asks.

## Satisfactory Style

- Clean white or warm-white background with a light financial editorial feel.
- Small top series bar and fine line, but not a PPT title page.
- Each page has a main headline, short explanation, chart/card/table layer, and a delayed conclusion bar.
- It is key information animation, not full subtitles.
- Cards, charts, K-line panels, rule tables, and conclusion bars enter according to narration meaning.
- The dark conclusion bar appears only when the voiceover reaches the conclusion.
- Bottom mini navigation may appear, but it must not occupy the Douyin caption/player safe zone.
- Book/product covers appear naturally, usually when the narration mentions the book/product or reaches the final CTA.
- Professional terms must be corrected from the user draft/PDF. ASR text is timing evidence only.

## Book Cover And Ending Lead

The book cover is part of the ending's trust and lead-generation structure. It should not be treated as a random product image.

For endings with a hook or traffic guidance:

- The ending should answer the opening pain point again before showing the book.
- The book cover should appear after the narration introduces the source/book/tool, then stay visible long enough to be recognized.
- Pair the cover with 2-3 useful viewer-facing tags, such as `回测表`, `指标验证`, `K线形态`, `概率方法`.
- Add a soft action only when it exists in the narration: `可以搜来看看`, `先从验证表开始`, `想系统看就从这本开始`.
- Avoid aggressive sales language unless the user explicitly wrote it.
- Never put internal labels such as `书名登场`, `背书推荐`, `行动指令`, or `CTA` on screen.

## Page Structure

A typical page uses:

- Top: audience-facing series theme, such as `裸K判断 | VA分型` or `炒股方法验证 | 少走弯路`.
- Small label: what the viewer is currently seeing, such as `指标越多越犹豫` or `规则要能被测试`.
- Main headline: one strong information point.
- Subcopy: one short explanation of the page.
- Middle: chart, card, K-line, comparison panel, rule table, metric card, or book card.
- Conclusion bar: appears near the spoken conclusion.
- Bottom: small navigation words the viewer understands, such as `规则`, `数据`, `验证`, `回归K线`.

## Animation Rules

Animation is triggered by narration semantics:

1. New idea starts: page title/headline appears.
2. Explanation starts: chart or cards appear.
3. Example or number is spoken: K-line, exact metric, table row, or comparison card appears.
4. Judgment/summary is spoken: dark conclusion bar appears.
5. Narration moves to the next idea: page transitions.

Never reveal all cards at the start of the scene. Never reveal the conclusion bar before the spoken conclusion.

## Information Density

A good page usually has:

- One main headline.
- One short explanatory line.
- Two to four visual elements.
- One conclusion bar.

The viewer should understand the page at a glance without reading a paragraph.

## Absolute Don'ts

Never put production labels on screen, including:

- `书名登场`
- `背书推荐`
- `行动指令`
- `人物登场`
- `第一篇`
- `第二篇`
- `第三篇`
- `动画素材`
- `信息动画素材`
- `同步预览`
- `预览`
- `静音版`
- `素材区`
- `成品区`

Do not use ASR as final on-screen copy. Use the user's script, PDF, or approved draft as the source of truth.

Do not create a single static page with a light push-in. The user has explicitly rejected that style.

## Multiple Audio Files

If the user provides multiple audio files with names that look like part 1 / part 2 of the same article, default to merging them into one final video unless the user explicitly asks for separate outputs.

- Is every screen phrase written from the viewer's perspective?
- Does every animation entrance correspond to voiceover meaning?

- Is the final output location reserved for final MP4s only, not review images or intermediates?
